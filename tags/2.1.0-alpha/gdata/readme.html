GData Java Client Library version 2.1.0-alpha

See: http://code.google.com/p/gdata-java-client/wiki/Version2

This library supports all Java 2 Standard Edition environments, including Android and App Engine.

Source Code and JavaDoc

All source code is included in gdata-2.1.0-alpha-src.zip . All Javadoc can be found in the javadoc folder.

Android

When building an application for Android, you will want to minimize the size of your application by only including the jars you need from this library in the android folder.

  • gdata-core.jar: required
  • gdata-xml.jar: needed for user-defined XML data models
  • gdata-json.jar and jackson-core-asl-1.5.0.jar: needed for user-defined JSON data models

App Engine and all other Java environments

When building an application for all other Java environments (other than Android), simplify your build process by just using the all-in-one jar. Dependent jars are found in the dependencies folder.

  • gdata-2.1.0-alpha.jar: required
  • jackson-core-asl-1.5.0.jar: needed for user-defined JSON data models
  • kxml2-2.3.0.jar: needed for user-defined XML data models
  • commons-logging-1.1.1.jar, httpcore-4.0.1.jar, and httpclient-4.0.1.jar: needed only if you prefer to use the Apache HTTP library for the underlying HTTP transport (default is to use the built-in java.net package of the JDK). Warning: Do not use the Apache HTTP library on App Engine because it won't work.
