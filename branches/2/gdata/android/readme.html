Android Client Library for Google API's version 2.3.0-alpha

This version 2 library is packaged in two different ways:

  • (this download) Android (gdata-android-2.3.0-alpha.zip)
  • All other Java 5 environments, including App Engine (gdata-java-2.3.0-alpha.zip)

See also the JavaDoc.

If you encounter a problem with this client library or want to request a feature, please submit an issue.

For discussions related to this client library only, please visit GData Java Client Group.

WARNING: this is an experimental branch of the library, and as such the client library API is subject to change. There are known bugs, and important features not yet implemented. We invite you to experiment with this library and provide us with early feedback.

Samples

  • Google Buzz API (JSON)
  • Picasa Web Albums Data API (Atom XML)

Library Jars

Minimum Android SDK supported: 1.5.

While developing an Android application, it is convenient to use gdata-2.3.0-alpha.jar from the Java download (gdata-java-2.3.0-alpha.zip). Unfortunately, its large size makes it unwieldy, especially since you are likely to only use a relatively small subset of the library. If multiple Android applications use this library on an Android device with limited application space, the problem grows significantly. Thus, to minimize the library size, use the packaged jars When building an application for Android. You can figure out the package dependencies by looking at end of the package description in the JavaDoc. Note that package dependencies may change in future versions, so beware of such changes when upgrading to a newer version.

Note: jackson-core-asl-1.5.2.jar is a third-party library needed when working with JSON. It can be downloaded from here.
